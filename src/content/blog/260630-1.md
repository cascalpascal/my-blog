# 여러 기기에서 블로그를 관리하기

이전까진 ubuntu를 사용했지만 다양한 개발 환경에 익숙해질 필요성을 느껴, Windows와 macOS 기기를 들였습니다.

따라서 다수의 기기에서 한 블로그를 운영하며 Vercel 기반의 CI/CD를 관리하기 위한 Git 루틴을 정리합니다.

### 기기 A에서 작업 끝낼 때
로컬 환경에서의 변경 사항을 GitHub 저장소에 기록하고 Vercel 서버가 자동으로 빌드 및 배포.

~~~bash
git add .
git commit -m "작업 완료"
git push
~~~

### 기기 B에서 작업 시작할 때
Github의 최신 상태를 로컬 환경에 동기화

~~~bash
git pull
~~~

### 기기 B에서 작업 끝낼 때
마찬가지로 GitHub에 백업

~~~bash
git add .
git commit -m "작업 완료"
git push
~~~
