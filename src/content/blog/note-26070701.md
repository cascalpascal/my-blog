## 데이터 링크 계층
**데이터 링크 계층(Layer 2)**: 물리적 링크를 통해 데이터를 전송

- 프레이밍: Layer 3에서 받은 패킷을 프레임으로 변환

- 흐름 제어: 송신 측의 데이터 전송 속도가 수신 측의 처리 속도보다 빠를 때 발생하는 데이터 손실을 방지

- 오류 제어: 전송 과정에서 데이터 오류 감지 및 수정 기능

### 프로토콜
Point to Point 데이터 전송 관리 및 오류 수정

#### HDLC(High-level Data Link Control)

- Bit 중심 프로토콜

- Point to Point, Multi Point 링크 모두 지원

- 프레임 구조: 헤더(플래그, 주소부, 제어부) - 패킷 - 트레일러(Frame Check Sequence, 플래그)

- Bit Stuffing: 플래그와 플래그가 아닌 부분을 구분하기 위해 사용

| HDLC 프레임 | 제어부 시작 | 역할 |
| :--- | :---: | --- |
| **I**(Information) | 0 | 데이터 전달 |
| **S**(Supervisory) | 10 | 오류·흐름 제어 |
| **U**(Unnumbered) | 11 | 링크 동작모드 설정 |

| HDLC 동작 모드 | 허가 여부 | 균형 여부 |
| :--- | :---: | --- |
| 정규 응답(NRM) | O | 한 개의 주국과 여러 개의 종국 |
| 비동기 응답(ARM) | X | Point to Point 불균형 링크 |
| 비동기 균형(ABM) | X | Point to Point 균형 링크 |

#### ATM(Asynchronous Transfer Mode)

- 53바이트 크기의 셀 단위로 데이터 전송

- 비동기식 시분할 다중화 방식(필요할 때, 시간 나눠, 여러 사용자 전송)


#### PPP(Point to Point Protocol)

- 문자 중심 프로토콜

- 두 노드 직접 연결 시 사용

- 오류 감지만 수행, 오류 복구는 상위 계층이

#### Frame Relay

- 고속 데이터 전송 프로토콜

- 중계 및 다중화 기능만 수행

### 데이터 링크 오류 제어

#### FEC(Forward Error Correction)

- 수신 측에서 오류를 스스로 감지하고 수정

- 해밍 코드: 1비트 오류 검출 및 **수정**

- 실시간 통신이나 단방향 통신에 적합

#### BEC(Backward Error Correction)

- 오류 발생 시 송신 측에 재전송 요구

- 패리티 검사: 패리티 비트(1비트)를 추가해 오류 검출. **수정 불가**

- 순환 중복 검사(CRC): 다항식 코드로 오류 검출

- 자동 반복 요청(ARQ): 재전송 기반 오류제어. 
    - Stop and wait: 한 프레임 전송 후 확인(ACK) 받기 기다림
    - Go Back N: 오류가 발생한 프레임부터 모두 재전송
    - Selective Repeat: 오류가 발생한 프래임만 재전송

- 블록 합 검사: 2차원 패리티 검사
