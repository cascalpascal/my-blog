## 네트워크 계층
**Layer 3(네트워크)**: 데이터의 경로를 설정(라우팅) 및 IP 주소 관리

### 프로토콜

#### IP(Internet Protocol)

- 데이터 송수신을 위한 주소 지정

- 데이터를 패킷 또는 데이터그램 단위로 캡슐화해 전송

- **비연결성**: 데이터를 전송하기 전에 연결 설정하지 않음

- **비신뢰성**: 패킷의 도착 순서나 전송 여부를 보장하지 않음

#### ARP(Address Resolution Protocol)

- 논리 주소(IP) &rarr; 물리 주소(MAC)로 변환

- 요청: 송신 측은 특정 IP 주소에 해당하는 MAC 주소를 찾기 위해 네트워크 전체에 ARP 요청 메시지를 브로드캐스트 함

- 응답: 해당 IP 주소를 사용하는 장비는 자신의 MAC 주소를 담아 ARP 응답 메시지를 유니캐스트로 송신 측으로 보냄

- 캐시: 송신 측은 응답받은 MAC 주소를 ARP 캐시에 일정 시간 저장

#### RARP(Reverse Address Resolution Protocol)

- 물리 주소(MAC) &rarr; 논리 주소(IP)로 변환

- 현재는 DHCP(Dynamic Jost Configuration Protocol)가 대체

#### ICMP(Internet Control Message Protocol)

- 네트워크 상태를 진단하기 위한 제어 메시지 전송

- IP는 오류 보고 기능이 없기 때문에 패킷 처리 시 발생하는 문제를 ICMP로 보고함

- Ping of Death에서 사용

#### IGMP(Internet Group Management Protocol)

- 멀티캐스트 그룹 관리

- 라우터는 IGMP를 통해 어떤 호스트가 어떤 멀티캐스트 그룹에 속해 있는지 파악하고, 해당 그룹으로 향하는 트래픽만 선택적으로 전달

#### NAT(Network Address Translation)

- 사설 IP 주소 &rarr; 공인 IP 주소 변환

- IPv4 주소 고갈 문제 해결

- 내부 네트워크를 외부로부터 보호
